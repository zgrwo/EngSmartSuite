## 分析决策树

```
拿到数据后，先问自己三个问题：

1. 我要做什么？
   ├── 找出影响质量的关键因子 → 场景一 (要因分析)
   ├── 找到最优参数组合 → 场景二 (DOE/优化)
   └── 监控过程是否稳定 → 场景三 (过程控制)

2. 我的数据长什么样？
   ├── 有明确的目标列 (Y) + 多个因子列 (X) → 相关性分析 → 回归
   ├── 有分组列 (如材料类型、机台号) + 测量值 → ANOVA / 假设检验
   ├── 只有一列测量值 → 分布摘要 / 正态性检验 / 异常检测
   └── 有时间顺序的连续测量 → 趋势预测 / SPC 控制图

3. 我的因子是什么类型？
   ├── 全是数值 (温度、压力、时间) → 相关性 / 回归 / 响应面
   ├── 全是类别 (材料、机台、班次) → ANOVA / 列联表 / 卡方检验
   └── 数值 + 类别混合 → 先做 ANOVA 筛类别因子，再做回归建模数值因子
```

## 场景一：出了质量问题，要找原因

### 工作流 1.1：快速扫描所有因子

**适用场景**：刚拿到一批数据，不知道哪个因子是罪魁祸首。

**步骤**：
1. **相关性分析** (`correlation`) — 看每个数值因子与目标的相关系数
2. **VIF 共线性诊断** (`vif`) — 排除"假相关"（两个参数实际是同一个东西）
3. **正态性评估** (`normality_check`) — 了解数据分布特征

**示例**：
```yaml
# step1_correlation.yaml
task: correlation
target_col: "不良率"
feature_cols:
  - "熔体温度"
  - "模具温度"
  - "注射压力"
  - "注射速度"
  - "冷却时间"
params:
  method: "pearson"
```

### 工作流 1.2：判断类别因子是否有影响

**适用场景**：想知道"换材料会不会改善质量？""夜班质量是不是比白班差？"

| 你的数据 | 分析方法 | Task Key |
|----------|---------|----------|
| 2 组对比 (新旧工艺) | 假设检验 t 检验 | `hypothesis_test` + `test: ttest_ind` |
| 3+ 组对比 (多种材料) | ANOVA | `anova` |
| 数据不正态时 | Kruskal-Wallis | `hypothesis_test` + `test: kruskal_wallis` |
| 前后测量对比 | 配对 t 检验 | `hypothesis_test` + `test: ttest_paired` |

### 工作流 1.3：深入建模 — 量化每个因子的影响

**适用场景**：已经知道哪些因子重要，想知道"温度升高 1°C，强度增加多少？"

**步骤**：
1. **回归分析** (`regression`) — 建立 Y = f(X₁, X₂, ...) 公式
2. **决策树** (`decision_tree`) — 辅助理解因子之间的交互关系
3. **检查模型质量** — 看 R²、残差诊断图、VIF

## 场景二：要找最优工艺参数

### 工作流 2.1：两个关键参数的联合优化

**适用场景**："温度和压力调到多少，强度最高？"

1. **响应面分析** (`response_surface`) — 生成 3D 曲面图和 2D 等高线
2. 看"山头"在哪 — 红色五角星标记了最优区域
3. 如果曲面平坦 → 参数在这个范围内影响不大，可以放宽控制

### 工作流 2.2：多个参数精确寻优

**适用场景**："我有 4 个参数可以调，每个都有范围，哪个组合最好？"

1. **回归分析** (`regression`) — 先建立预测模型
2. **网格搜索** (`grid_search`) — 在参数范围内自动找到最优组合
3. 查看 Top 5 候选方案

### 工作流 2.3：多个目标需要同时优化

**适用场景**："既要强度高、又要不良率低、还要成本可接受"

1. **多目标优化** (`multi_objective`) — 加权期望函数法
2. 设置每个目标的优化方向 (最大化/最小化) 和权重
3. 查看综合评分排名和 Pareto 前沿

### 工作流 2.4：分析正式 DOE 实验数据

**适用场景**：已经按 DOE 矩阵做过实验，需要分析效应。

1. **DOE 效应分析** (`doe_analysis`) — Pareto 图显示各因子的主效应
2. **Lenth PSE 参考线** — 判断效应是否显著（红色虚线）

## 场景三：要持续监控过程

### 工作流 3.1：判断过程是否稳定

**适用场景**："每天抽检 5 个样品，想画控制图。"

1. **SPC 控制图** (`spc_xbar`) — X-bar + R 图
2. 检查违规点（红色圆圈标记）
3. 看 Western Electric 规则触发情况

**控制图解读**：

| 违规类型 | 含义 | 可能原因 |
|---------|------|---------|
| 点超出 UCL/LCL | 过程失控 | 设备故障、原料异常、操作失误 |
| 连续 7 点同侧 | 过程均值已偏移 | 模具磨损、传感器漂移 |
| 连续 6 点趋势 | 过程在缓慢漂移 | 刀具磨损、温度逐渐升高 |
| 连续 15 点在 ±1σ 内 | 控制限过宽/分层 | 子组划分不合理 |

### 工作流 3.2：评估能否满足客户规格

**适用场景**：客户审核要求 Cpk ≥ 1.33。

1. **过程能力分析** (`process_capability`) — 设置 USL/LSL
2. 查看 Cpk 值和判定
3. 如果数据不正态 → 使用 `transform: boxcox`

**Cpk 判定标准**：

| Cpk | 判定 | 建议 |
|-----|------|------|
| ≥ 1.67 | 优秀 ✓ | 可以考虑放宽抽检频率 |
| 1.33 ~ 1.67 | 合格 ✓ | 行业通用标准，正常控制 |
| 1.0 ~ 1.33 | 需改进 ⚠ | 不良品风险较高，需改善 |
| < 1.0 | 不合格 ✗ | 必须立即改进 |

### 工作流 3.3：预测未来趋势

**适用场景**："参数一直在慢慢往上漂，再过多久会超标？"

1. **趋势预测** (`trend_forecast`) — 设置预测步数
2. 查看预测区间（橙色区域）
3. 检查 Durbin-Watson 和 Ljung-Box 检验

### 工作流 3.4：发现异常点

**适用场景**："这批数据有几万个，想快速找出不对劲的。"

| 方法 | 适用场景 | 参数 |
|------|---------|------|
| IQR 法 | 单变量，快速筛查 | `method: iqr` |
| Z-score 法 | 数据接近正态 | `method: zscore` |
| Grubbs 检验 | 严格统计检验 | `method: grubbs` |
| Isolation Forest | 多变量联合异常 | `method: isolation_forest` |
| 多方法共识 | 减少误报 (≥2 票) | `outlier_consensus` |

## 结果解读指南

### 相关性分析

| 你看到 | 含义 | 怎么做 |
|--------|------|--------|
| `r = 0.312，与「不良率」最相关的是「熔体温度」` | 熔体温度每升高 1 个标准差，不良率上升约 0.312 个标准差 | 优先调整熔体温度做验证实验 |
| `r ≈ 0` | 该因子与结果没有线性关系 | 排除该因子，或尝试非线性方法（响应面） |
| `p 值 < 0.05` | 相关性在统计上可信 | 可以据此做决策 |
| `p 值 > 0.05` | 相关性可能是随机波动 | 需要更多数据确认 |
| Bonferroni 校正后显著数变化 | 原始 p 值可能高估了显著性 | 以校正后为准 |

### ANOVA

| 你看到 | 含义 | 怎么做 |
|--------|------|--------|
| `原料类型(p=0.0000) 对冲击强度有显著影响` | 不同材料之间的差异不是偶然的 | 换材料是有效的改善方向 |
| `未发现显著影响的因子 (α=0.05)` | 在你给的数据中，没有因子能解释 Y 的变化 | 检查数据量是否充足，或因子范围太窄 |
| `R² = 0.82` | 模型能解释 82% 的变差 | 模型拟合很好，可以用于预测 |
| `R² = 0.12` | 模型只能解释 12% 的变差 | 可能有重要的因子没被测量 |
| `η² = 0.35 (大)` | 该因子的效应量很大 | 这是主要影响因素 |
| Levene p < 0.05 | 方差齐性不满足 | ANOVA 结果需谨慎解读，考虑非参数检验 |
| Shapiro-Wilk p < 0.05 | 残差不正态 | 考虑数据变换或使用 Kruskal-Wallis |

### 假设检验

| 你看到 | 含义 | 怎么做 |
|--------|------|--------|
| `旧工艺 vs 新工艺: 存在显著差异 (p=0.0019)` | 差异不是偶然的 | 确认差异方向，决定是否切换 |
| `未发现显著差异 (p=0.42)` | 数据没有足够证据说明两者不同 | 可能确实没差异，也可能是样本量太小 |
| `效应量 Cohen's d = 0.85 (大)` | 两组差异很大 | 即使 p 值不显著也应该关注 |
| `效应量可忽略` | 即使统计显著，实际差异很小 | 工程意义不大 |
| `统计功效 0.45` | 样本量太小，即使有差异也检测不出来 | 增加样本量 |

### 回归建模

| 你看到 | 含义 | 怎么做 |
|--------|------|--------|
| `料温 系数=0.06` | 料温每升高 1°C，强度平均增加 0.06 MPa | 用于设定控制范围 |
| `标准化系数(β) 最大` | 该变量的相对影响力最大 | 优先控制 |
| `Durbin-Watson 接近 2` | 残差无自相关 ✓ | 模型假设满足 |
| `Breusch-Pagan p < 0.05` | 存在异方差 | 考虑使用稳健回归或变量变换 |
| `Cook's D > 4/n` | 存在强影响点 | 检查该数据点是否正确 |

### 过程能力

| Cpk 值 | 判定 | Sigma Level | DPMO (每百万缺陷数) |
|--------|------|-------------|---------------------|
| 2.00 | 世界级 | 6σ | 3.4 |
| 1.67 | 优秀 | 5σ | 233 |
| 1.33 | 合格 | 4σ | 6,210 |
| 1.00 | 临界 | 3σ | 66,807 |
| 0.67 | 不足 | 2σ | 308,538 |

### 生存分析

| 你看到 | 含义 | 怎么做 |
|--------|------|--------|
| 中位寿命 = 1200 小时 | 50% 的产品在 1200 小时内失效 | 制定维保周期 |
| Weibull β > 1 | 失效率随时间上升 (磨损) | 定期更换 |
| Weibull β = 1 | 随机失效 | 不需要预防性维护 |
| Weibull β < 1 | 早期失效 (浴盆曲线) | 加强出厂检验 |
| Log-rank p < 0.05 | 两组寿命有显著差异 | 选择更可靠的方案 |

## 模板使用指南

### 模板结构

```yaml
task: anova                  # 分析方法 (必需)
target_col: "不良率"         # 目标列名 (必需)
feature_cols:                # 因子列名 (可选)
  - "料温"
  - "模温"
params:                      # 方法参数 (可选)
  alpha: 0.05
```

### 常用模板示例

**自动假设检验** (自动选择 t 检验或 Mann-Whitney)：
```yaml
task: hypothesis_test
target_col: "强度"
feature_cols: ["工艺"]
params:
  test: "auto"
  group_col: "工艺"
```

**过程能力分析** (含规格限)：
```yaml
task: process_capability
target_col: "尺寸"
params:
  usl: 10.5
  lsl: 9.5
  target: 10.0
```

**响应面优化**：
```yaml
task: response_surface
target_col: "拉伸强度"
feature_cols: ["熔体温度", "注射压力"]
params:
  direction: "maximize"
```

## 排错 FAQ

### Q: 分析返回 "有效数据不足"
**原因**: 数据行数太少或包含大量缺失值。
**解决**: 确保至少有 10 行以上有效数据。检查 Excel 中是否有空行或合并单元格。

### Q: 分析返回 "不是二分类数据"
**原因**: 目标列包含超过 2 个不同的值（如 "合格/不合格/待定"）。
**解决**: 筛选数据只保留两种类别，或使用 ANOVA 分析多类别。

### Q: 图表中文显示为方块
**原因**: matplotlib 未找到中文字体。
**解决**: 安装 Microsoft YaHei 字体，或在 `engine/__init__.py` 中配置其他中文字体路径。

### Q: ANOVA 返回 "残差不满足正态性假设"
**原因**: 数据分布严重偏态或存在异常值。
**解决**: 使用 `normality_check` 查看各列正态性；考虑数据变换 (Box-Cox)；或使用 Kruskal-Wallis 非参数检验。

### Q: Lasso 回归选中了 0 个变量
**原因**: α 值过大，或所有变量与目标的相关性都很弱。
**解决**: 减小 `alpha_lasso` 值，或先用相关性分析确认变量与目标的关系。

### Q: 过程能力 Cpk 显示 "N/A"
**原因**: 未提供规格限 USL/LSL。
**解决**: 在参数中设置 `usl` 和 `lsl`。

### Q: 散点矩阵或 3D 图未生成
**原因**: 样本量不足 (散点矩阵需要 ≥20 个点才会绘制 LOWESS 趋势线)。
**解决**: 增加数据量，这不影响数值分析结果。

## 最佳实践

### 数据准备
1. **第一行为表头** — Excel 数据第一行必须是列名
2. **避免合并单元格** — 合并单元格会导致读取错误
3. **缺失值统一留空** — 不要填 "N/A" 或 "-"，直接留空
4. **数值列不要混入文字** — "温度" 列中出现 "约200" 会导致整列被当作文本
5. **类别列用中文标签** — "白班/夜班" 比 "A/B" 更直观

### 分析流程
1. **先看数据质量** — 用 `normal_check` + `distribution_summary` 了解数据概况
2. **再扫相关性** — `correlation` 快速识别候选因子
3. **建模前查共线性** — `vif` 排除假相关
4. **多种方法交叉验证** — 用 `decision_tree` 验证 `correlation` 的发现
5. **结论要回溯到工艺** — 统计显著不等于工程显著，要结合工艺经验判断

### 报告输出
1. **日常分析** → Excel 输出（数据和图表在同一文件）
2. **周报/月报** → PPT 输出（适合会议汇报）
3. **审核/归档** → PDF 输出（格式固定，不可修改）
4. **分享给无 Excel 的同事** → HTML 输出（浏览器打开即可）
